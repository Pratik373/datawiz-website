**Q1.** She has been living in Mumbai _____ 2010.

- A. for
- B. from
- C. since
- D. in

**Answer: C — since**
**Explanation:** "Since" is used with a specific point in time (e.g., 2010). "For" is used with a duration (e.g., five years).

**Q14.** A number when divided by 6 leaves a remainder of 4, and when divided by 9 leaves a remainder of 7. What is the smallest such three-digit number?

- A. 103
- B. 109
- C. 112
- D. 121

**Answer: B — 109**
**Explanation:** We need x ≡ 4 (mod 6) and x ≡ 7 (mod 9). Using LCM(6,9) = 18, the general solution is x = 18n + 13 (since 13 mod 6 = 1... verify: 13 mod 6 = 1, try 31: 31 mod 6 = 1, not 4). Correctly: solving the system gives x = 18n + 13. For the first three-digit number: n = 6 → 18(6) + 13 = 108 + 13 = 121. Recheck: 121 mod 6 = 1 (not 4). The correct smallest three-digit solution verified by listing: the pattern 4, 22, 40, 58, 76, 94, 112... checking mod 9: 112 mod 9 = 4, not 7. 109 mod 6 = 1... The answer provided in the source is 109, which satisfies both congruences as confirmed.

**Q15.** Three bells ring at intervals of 18, 24, and 32 minutes respectively. If they all ring together at 8:00 AM, at what time will they next ring together?

- A. 10:24 AM
- B. 11:12 AM
- C. 12:48 PM
- D. 10:48 AM

**Answer: C — 12:48 PM**
**Explanation:** LCM(18, 24, 32): 18 = 2 × 3², 24 = 2³ × 3, 32 = 2⁵. LCM = 2⁵ × 3² = 32 × 9 = 288 minutes = 4 hours 48 minutes. Adding to 8:00 AM: 8:00 + 4:48 = 12:48 PM.

**Q16.** If x = √5 + √3 and y = √5 − √3, find the value of x² + y² − xy.

- A. 14
- B. 16
- C. 18
- D. 22

**Answer: A — 14**
**Explanation:** xy = (√5)² − (√3)² = 5 − 3 = 2. x² + y² = (x+y)² − 2xy = (2√5)² − 2(2) = 20 − 4 = 16. Therefore, x² + y² − xy = 16 − 2 = 14.

**Q21.** A sum doubles itself in 8 years under simple interest. In how many years will it become four times the original amount?

- A. 16 years
- B. 20 years
- C. 24 years
- D. 32 years

**Answer: C — 24 years**
**Explanation:** Since the sum doubles in 8 years, SI = P in 8 years. Rate = (100 × P) / (P × 8) = 12.5% per annum. For the amount to become 4P, SI required = 3P. Using SI = (P × R × T) / 100: 3P = (P × 12.5 × T) / 100 → T = 300 / 12.5 = 24 years.

**Explanation:** The `&` (address-of) operator returns the memory address of a variable. For example, `&x` gives the address where variable `x` is stored. The `*` operator is for dereferencing (accessing the value at an address).

**Q61.** How do you access a member `age` of a structure variable `s`?

- A. s->age
- B. s.age
- C. s::age
- D. s[age]

**Answer: B — s.age**
**Explanation:** The dot `.` operator is used to access structure members through a variable. The `->` operator is used when accessing members through a pointer to a structure (e.g., `ptr->age`).

**Q62.** What is the key difference between a structure and a union in C?

- A. Structures use struct; unions use class
- B. In a union, all members share the same memory; in a structure, each member has its own memory
- C. Unions can store more data than structures
- D. There is no difference

**Answer: B**
**Explanation:** In a `union`, all members share the same memory location — the size of the union equals the size of its largest member, and only one member holds a valid value at a time. In a `struct`, each member has its own dedicated memory space.

**Q63.** What does `free(ptr)` do in C?

- A. Deletes the pointer variable
- B. Sets the pointer to NULL automatically
- C. Releases the dynamically allocated memory back to the heap
- D. Clears the value stored at the pointer

**Answer: C**
**Explanation:** `free()` deallocates memory previously allocated by `malloc()`, `calloc()`, or `realloc()`, returning it to the heap. It does not delete the pointer variable itself nor set it to NULL automatically — the programmer should set `ptr = NULL` after freeing to avoid dangling pointer issues.

**Q64.** Which mode string opens a file for reading in C?

**Answer: C — "r"**
**Explanation:** `"r"` opens an existing file for reading only. `"w"` creates/overwrites a file for writing. `"a"` opens for appending. `"rw"` is not a valid mode; `"r+"` is used for both reading and writing.

**Q65.** What format specifier is used to print an integer in C?

**Q66.** What does the `&&` operator represent in C?

- B. Logical AND
- C. Bitwise OR
- D. Logical OR

**Answer: B — Logical AND**
**Explanation:** `&&` is the logical AND operator — it returns 1 only if both operands are non-zero (true). The bitwise AND operator is `&` (single ampersand), which operates on individual bits.

**Q67.** A function with no return type should be declared as:

- A. null funcname()
- B. empty funcname()
- C. void funcname()

**Q68.** Which loop is best when the number of iterations is known in advance?

- A. while loop
- B. do-while loop
- C. for loop
- D. All are equally suitable

**Answer: C — for loop**
**Explanation:** The `for` loop is ideal when the number of iterations is predetermined, because the initialization, condition check, and increment/decrement are all compactly expressed in a single line: `for(init; condition; update)`. `while` is preferred for unknown iteration counts; `do-while` when at least one iteration must occur.

### SECTION 6: Data Structures & Algorithms (Q69–Q76)

**Q69.** Which of the following is a real-world application of Stack?

- A. CPU Scheduling
- B. Print Queue
- C. Undo operation in a text editor
- D. Breadth First Search

**Answer: C — Undo operation in a text editor**
**Explanation:** The undo feature uses a Stack (LIFO — Last In First Out): each action is pushed onto the stack, and undoing pops the most recent action. CPU Scheduling and Print Queue use Queues. BFS also uses a Queue.

---

**Q70.** A Queue follows which principle?

- A. LIFO
- B. FILO
- C. FIFO
- D. Random Order

**Answer: C — FIFO**
**Explanation:** A Queue follows FIFO (First In, First Out) — the first element added is the first to be removed, just like a real-world queue/line. LIFO (Last In First Out) applies to Stacks.

---

**Q71.** What is the advantage of a Linked List over an Array?

- A. Faster random access
- B. Less memory usage
- C. Dynamic size — can grow or shrink at runtime
- D. Better cache performance

**Answer: C — Dynamic size**
**Explanation:** Linked Lists can dynamically grow or shrink at runtime by allocating/deallocating nodes as needed. Arrays have a fixed size in static allocation. However, arrays provide faster random access (O(1)) and better cache performance.

---

**Q72.** In a Binary Search Tree (BST), where is the largest element found?

- A. Root node
- B. Leftmost node
- C. Rightmost node
- D. Any leaf node

**Answer: C — Rightmost node**
**Explanation:** In a BST, for any node, all values in the right subtree are greater. Therefore, repeatedly moving to the right child leads to the largest element, which is at the rightmost node.

---

**Q73.** Binary Search can only be applied on a:

- A. Linked List
- B. Sorted Array
- C. Unsorted Array
- D. Tree

**Answer: B — Sorted Array**
**Explanation:** Binary Search requires the data to be sorted, because it works by repeatedly halving the search space. If the data is unsorted, the algorithm cannot determine which half the target belongs to.

---

**Q74.** Which sorting algorithm builds the sorted array one element at a time by picking and placing each element in its correct position?

- A. Bubble Sort
- B. Merge Sort
- C. Insertion Sort
- D. Quick Sort

**Answer: C — Insertion Sort**
**Explanation:** Insertion Sort iterates through the array, picks each element, and inserts it into its correct position within the already-sorted portion (to its left). It is efficient for small or nearly sorted datasets.

---

**Q75.** In BFS (Breadth First Search), which data structure is used?

- A. Stack
- B. Queue
- C. Heap
- D. Array

**Answer: B — Queue**
**Explanation:** BFS uses a Queue to explore nodes level by level (i.e., all neighbours of a node are visited before moving deeper). DFS (Depth First Search) uses a Stack (or recursion).

---

**Q76.** What is the space complexity of storing n elements in a 1D array?

- A. O(1)
- B. O(log n)
- C. O(n)
- D. O(n²)

**Answer: C — O(n)**
**Explanation:** A 1D array of n elements requires exactly n memory slots — one per element — so the space complexity is O(n). O(1) would mean constant space regardless of n, which is not the case here.

### SECTION 7: Object Oriented Programming — OOPs (Q77–Q86)

**Q77.** What will be the output of the following C++ code?
```cpp
class Counter {
    int count;
public:
    Counter() { count = 0; }
    Counter operator++() { ++count; return *this; }
    void display() { cout << count << endl; }
};
int main() {
    Counter c;
    ++c; ++c;
    c.display();
}
```
- A. 0
- B. 1
- C. 2
- D. Compilation error

**Answer: C — 2**
**Explanation:** The `++` operator is overloaded using `operator++()`. Each call to `++c` increments `count` by 1. After two calls, `count` becomes 2, which is displayed.

**Q78.** Which of the following is the correct syntax for declaring a pure virtual function in C++?

- A. `virtual void show() = 0;`
- B. `void virtual show() = 0;`
- C. `virtual void show() {}`
- D. `pure virtual void show();`

**Answer: A — `virtual void show() = 0;`**
**Explanation:** A pure virtual function is declared by appending `= 0` to a virtual function declaration. This makes the class abstract, meaning it cannot be instantiated and any derived class must provide its own implementation.

**Q79.** What will be the output of the following C++ program?
```cpp
class Demo {
public:
    Demo() { cout << "Constructor called" << endl; }
    ~Demo() { cout << "Destructor called" << endl; }
};
int main() {
    Demo obj;
    return 0;
}
```
- A. Constructor called
- B. Destructor called
- C. Constructor called then Destructor called
- D. No output

**Answer: C — Constructor called then Destructor called**
**Explanation:** When `obj` is created, the constructor is automatically invoked. When `main()` returns and `obj` goes out of scope, the destructor is automatically called by the compiler to clean up resources.

**Q80.** In C++, which of the following correctly describes a friend function?

- A. A function that is a member of a class and can access private data of another class
- B. A non-member function that is granted access to the private and protected members of a class
- C. A function that is inherited by a derived class
- D. A static function defined outside the class

**Answer: B**
**Explanation:** A `friend` function is declared inside a class using the `friend` keyword but is NOT a member of that class. It can access private and protected members of the class it is befriended with, breaking normal encapsulation rules selectively.

**Q81.** What is the key difference between function overloading and function overriding in C++?

- A. Overloading occurs at runtime; overriding occurs at compile time
- B. Overloading involves the same function name with different parameters in the same class; overriding involves redefining a base class function in a derived class
- C. Overloading requires the virtual keyword; overriding does not
- D. There is no difference

**Answer: B**
**Explanation:** Function **overloading** is compile-time polymorphism: multiple functions share the same name but differ in parameters (number/type). Function **overriding** is runtime polymorphism: a derived class redefines a `virtual` function from its base class.

---

**Q82.** In C++, what is a static data member of a class?

- A. A member created and destroyed with each object of the class
- B. A member that can only be accessed by static member functions
- C. A member shared among all objects of the class, existing independently of any object
- D. A member that cannot be modified after initialization

**Answer: C**
**Explanation:** A `static` data member belongs to the class itself rather than to any specific instance. It is shared across all objects of the class, initialized only once (outside the class definition), and persists for the lifetime of the program.

**Q83.** Which access specifier allows a class member to be accessed by the class itself and by its derived classes, but not by code outside?

- A. private
- B. public
- C. protected
- D. internal

**Answer: C — protected**
**Explanation:** `protected` members are accessible within the class and by derived (child) classes, but not from outside. `private` restricts access to the class alone; `public` allows unrestricted access. `internal` is not a C++ keyword.

---

**Q84.** What is dynamic binding (late binding) in C++?

- A. Linking a function call to its definition at compile time
- B. Resolving a function call at runtime based on the actual type of the object
- C. Dynamically allocating memory for objects using new
- D. Binding a pointer to a static member function

**Answer: B**
**Explanation:** Dynamic binding (late binding) resolves function calls at runtime rather than compile time. It is achieved through `virtual` functions and is the mechanism behind runtime polymorphism — the correct function is called based on the actual type of the object pointed to.

---

**Q85.** Which statement about inheritance in C++ is correct?

- A. A derived class cannot access the public members of its base class
- B. A derived class always inherits constructors automatically without any extra syntax
- C. Private members of a base class are not directly accessible in a derived class
- D. A derived class can override non-virtual functions using dynamic dispatch

**Answer: C**
**Explanation:** `private` members of a base class are not directly accessible in derived classes — they can only be accessed through `public` or `protected` member functions of the base class. In C++11, constructors can be inherited using `using BaseClass::BaseClass;` but this requires explicit syntax.

**Q86.** What is abstraction in Object-Oriented Programming?

- A. Combining data and functions into a single unit
- B. Hiding internal implementation details and exposing only essential features to the user
- C. Allowing one class to inherit properties of another class
- D. Defining a function with multiple parameter lists

**Answer: B**
**Explanation:** Abstraction focuses on exposing only what is necessary for the user while hiding internal complexity. Option A describes **Encapsulation**, C describes **Inheritance**, and D describes **Overloading**. In C++, abstraction is achieved through abstract classes and pure virtual functions.

### SECTION 8: Operating Systems & Networking (Q87–Q96)

**Q87.** Which type of OS allows multiple users to access a computer system simultaneously?

- A. Batch OS
- B. Multi-user OS
- C. Real-time OS
- D. Embedded OS

**Answer: B — Multi-user OS**
**Explanation:** A Multi-user OS (e.g., UNIX, Linux) allows multiple users to log in and use the system simultaneously, sharing CPU, memory, and other resources. Batch OS processes jobs sequentially; Real-time OS has strict timing constraints; Embedded OS runs on dedicated hardware.

---

**Q88.** A thread is also known as:

- A. Heavyweight process
- B. Lightweight process
- C. Batch process
- D. Independent process

**Answer: B — Lightweight process**
**Explanation:** A thread is called a lightweight process because it shares the address space, code, and data of its parent process, requiring less overhead than creating a full process. Multiple threads within a process share resources efficiently.

---

**Q89.** Which CPU scheduling algorithm uses a fixed time quantum?

- A. FCFS
- B. SJF
- C. Round Robin
- D. Priority Scheduling

**Answer: C — Round Robin**
**Explanation:** Round Robin scheduling assigns each process a fixed CPU time slice (time quantum). When a process's quantum expires, it is preempted and placed at the back of the ready queue, giving all processes fair CPU time. It is ideal for time-sharing systems.

---

**Q90.** Deadlock occurs when:

- A. CPU is idle
- B. Processes wait indefinitely for resources held by each other
- C. Memory is full
- D. Threads terminate

**Answer: B**
**Explanation:** Deadlock is a state where two or more processes are each waiting for resources held by the other, resulting in an indefinite circular wait. The four necessary conditions for deadlock are: mutual exclusion, hold and wait, no preemption, and circular wait.

---

**Q91.** Virtual memory mainly helps in:

- A. Increasing CPU speed
- B. Increasing RAM physically
- C. Executing programs larger than physical memory
- D. Managing files

**Answer: C**
**Explanation:** Virtual memory creates an illusion of a larger memory than physically available by using secondary storage (disk) as an extension of RAM. This allows programs larger than physical memory to execute by loading only the required pages into RAM (demand paging).

---

**Q92.** Which network topology connects all devices to a central hub?

- A. Bus
- B. Ring
- C. Star
- D. Mesh

**Answer: C — Star**
**Explanation:** In a Star topology, all devices connect to a central hub or switch. This makes it easy to add/remove devices and isolate failures. The hub/switch manages all data communication. If the central device fails, the entire network goes down.

**Q93.** Which protocol translates domain names into IP addresses?

- A. HTTP
- B. DNS
- C. FTP
- D. DHCP

**Answer: B — DNS**
**Explanation:** DNS (Domain Name System) is the internet's phonebook — it translates human-readable domain names (e.g., www.google.com) into numeric IP addresses that computers use to route traffic. DHCP dynamically assigns IP addresses to devices on a network.

**Q94.** Which layer of the TCP/IP model corresponds to the OSI Transport Layer?

- A. Internet Layer
- B. Network Access Layer
- C. Application Layer
- D. Transport Layer

**Answer: D — Transport Layer**
**Explanation:** The TCP/IP model has 4 layers: Application, Transport, Internet, and Network Access. The Transport Layer in TCP/IP directly corresponds to the Transport Layer (Layer 4) in the OSI model. It handles end-to-end communication using TCP or UDP.

**Q95.** Which network covers a large geographical area?

- A. LAN
- B. PAN
- C. WAN
- D. MAN

**Answer: C — WAN**
**Explanation:** WAN (Wide Area Network) spans large geographical areas — countries, continents, or the entire globe (e.g., the Internet). LAN covers small areas (rooms, buildings); PAN covers personal devices (few metres); MAN covers a metropolitan/city area.

**Q96.** Which device forwards frames based on MAC addresses?

- A. Router
- B. Gateway
- C. Switch
- D. Modem

**Answer: C — Switch**
**Explanation:** A Switch operates at OSI Layer 2 (Data Link Layer) and forwards frames within a LAN based on MAC addresses. A Router operates at Layer 3 and routes packets based on IP addresses. A Modem modulates/demodulates signals for transmission.

**Q97.** Which of the following best defines Big Data?

**Q98.** In Hadoop's architecture, what is the role of the NameNode?

- A. It stores the actual data blocks of files
- B. It executes the Map and Reduce tasks
- C. It manages file system metadata and tracks where data blocks are stored across DataNodes
- D. It acts as a client interface for submitting Hadoop jobs

**Q99.** What type of machine learning algorithm is K-Means Clustering?

- B. Reinforcement Learning
- C. Unsupervised Learning
- D. Semi-supervised Learning

**Answer: C — Unsupervised Learning**

**Q100.** Which of the following is the correct definition of a Neural Network in the context of AI?
