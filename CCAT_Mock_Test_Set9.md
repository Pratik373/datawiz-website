**Q1.** The chairman's tone during the press conference was surprisingly _____, given the severity of the allegations against him.
- A) defensive
- B) aggressive
- C) nonchalant
- D) alarmed

**Answer:** C
**Explanation:** "Nonchalant" means casually calm and relaxed, which would be surprising given serious allegations — creating the contrast the sentence implies. "Defensive" or "alarmed" would be expected reactions; "aggressive" is possible but less ironic. Only "nonchalant" captures the unexpected calm.

**Q2.** The summit was convened _____ purpose of addressing the escalating climate crisis.
- A) with the
- B) for the
- C) at the
- D) by the

**Answer:** B
**Explanation:** "For the purpose of" is the correct prepositional phrase indicating the objective or reason behind an action. "With the purpose of" is informal but less standard. "At the purpose" and "by the purpose" are grammatically incorrect collocations.

**Q3.** Spot the error: "The jury has given their verdict after three days of deliberation."
- A) The jury has given
- B) their verdict
- C) after three days
- D) of deliberation

**Answer:** B
**Explanation:** "Jury" is a collective noun. In formal British English, collective nouns can take plural pronouns ("their"), but if the verb "has" (singular) is used, then the pronoun must also be singular ("its"). The mismatch between singular "has" and plural "their" makes this inconsistent. The correct form is either "The jury has given **its** verdict" or "The jury have given their verdict."

**Q4.** Fill in the blank: By the time the ambulance arrived, the patient _____ consciousness.
- A) loses
- B) lost
- C) had lost
- D) has lost

**Answer:** C
**Explanation:** When two past events are described and one happened before the other, the earlier event uses Past Perfect ("had lost"). The patient losing consciousness happened before the ambulance arrived — hence "had lost" is correct. Simple Past "lost" would suggest both events happened simultaneously.

**Q6.** Improve the sentence: "We discussed about the new policy in the meeting."

**Q8.** She _____ the violin since the age of six and has performed in three international concerts.
- A) plays
- B) played
- C) has been playing
- D) had played

**Answer:** C
**Explanation:** "Since the age of six" indicates an action that began in the past and continues into the present — requiring Present Perfect Continuous ("has been playing"). This tense emphasises the duration and continuity of the action. Simple Past "played" implies the action is over; "plays" (Simple Present) ignores the historical starting point.

**Q9.** Change to indirect speech: The teacher said, "Work hard and you will succeed."

- D) The teacher said that they should work hard and would succeed.

**Answer:** B
**Explanation:** Imperative sentences in direct speech become infinitive clauses ("advised/told someone to do") in indirect speech. "Will" shifts to "would" (backshift). - C) fgets(str, sizeof(str), stdin);
- D) Both B and C

**Answer:** C
**Explanation:** `scanf("%s")` stops reading at the first whitespace, so it cannot read strings with spaces. `gets()` reads until newline but is unsafe (no buffer overflow protection) and deprecated since C11. `fgets()` is the safe, recommended alternative — it reads up to n-1 characters including spaces, and null-terminates the string. In modern C, `fgets()` is the preferred choice.

**Q57.** What is a "dangling pointer" in C?

### Part 7: Data Structures (Q66–Q73)

**Q66.** What is the time complexity of inserting an element at the beginning of a singly linked list?
- A) O(n)
- B) O(log n)
- C) O(1)
- D) O(n²)

**Answer:** C
**Explanation:** Inserting at the beginning of a singly linked list requires only: creating a new node, setting its next pointer to the current head, and updating the head pointer. This takes constant time O(1), regardless of list length. Contrast this with arrays where insertion at the beginning requires shifting all elements O(n).

---

**Q67.** Which data structure is best suited for implementing the "Undo" feature in a text editor?
- A) Queue
- B) Stack
- C) Binary Search Tree
- D) Hash Table

**Answer:** B
**Explanation:** The "Undo" feature requires reversing the most recent action first — Last In, First Out (LIFO) behaviour. A Stack is ideal: each action is pushed; pressing Undo pops the most recent action. A Queue (FIFO) would undo the oldest action first, which is incorrect. BSTs and Hash Tables are for searching, not sequencing operations.

---

**Q68.** In a Binary Search Tree (BST), which traversal visits nodes in sorted (ascending) order?
- A) Preorder
- B) Postorder
- C) Inorder
- D) Level order

**Answer:** C
**Explanation:** Inorder traversal (Left → Root → Right) of a BST visits nodes in ascending sorted order. This property makes inorder traversal essential for BST operations. Preorder (Root → Left → Right) is used for copying trees. Postorder (Left → Right → Root) is used for deleting trees.

---

**Q69.** What is the maximum number of nodes in a binary tree of height h (where the root is at height 0)?
- A) 2h
- B) 2^h − 1
- C) 2^(h+1) − 1
- D) h²

**Answer:** C
**Explanation:** A complete binary tree of height h has 1 node at level 0 (root), 2 at level 1, 4 at level 2, ..., 2ʰ at level h. Total nodes = 2⁰ + 2¹ + ... + 2ʰ = 2^(h+1) − 1. For example, height 2: 1+2+4 = 7 = 2³−1 = 2^(2+1)−1 ✓.

---

**Q70.** Which of the following operations is NOT efficiently supported by a Hash Table?
- A) Insertion
- B) Search by key
- C) Deletion
- D) Finding the minimum element

**Answer:** D
**Explanation:** Hash tables provide O(1) average-case for insertion, search (by key), and deletion. However, finding the minimum (or maximum) element requires scanning all n entries O(n) because hash tables do not maintain sorted order. For min/max operations, a heap or sorted structure is more appropriate.

---

**Q71.** A queue is implemented using two stacks. What is the time complexity of the dequeue operation (amortised)?
- A) O(1)
- B) O(n)
- C) O(log n)
- D) O(n²)

**Answer:** A
**Explanation:** In the two-stack queue implementation, enqueue pushes to stack1 O(1). Dequeue: if stack2 is empty, move all elements from stack1 to stack2 (O(n) in worst case for one dequeue), then pop from stack2. Amortised over n operations, each element is moved at most once — making the amortised cost O(1) per dequeue.

**Q72.** What is "collision" in the context of hashing?
- A) When two processes access the same memory
- B) When two different keys produce the same hash value
- C) When a stack overflows
- D) When a linked list has a cycle

**Answer:** B
**Explanation:** A collision occurs when two different keys are mapped to the same index (bucket) by the hash function. Since hash functions map a large key space to a smaller table, collisions are inevitable. Common resolution techniques include chaining (linked lists at each bucket) and open addressing (linear probing, quadratic probing).

**Q73.** What is the space complexity of Merge Sort?
- A) O(1)
- B) O(log n)
- C) O(n)
- D) O(n log n)

**Answer:** C
**Explanation:** Merge Sort requires O(n) auxiliary space to create temporary arrays during the merge step. This is a disadvantage compared to in-place sorting algorithms like Heap Sort (O(1) space) or Quick Sort (O(log n) stack space for recursion). The O(n) space is required to merge two halves.

### Part 8: Object Oriented Programming (Q74–Q83)

**Q74.** Which OOP concept allows a class to have multiple methods with the same name but different parameters?

**Answer:** C
**Explanation:** Method Overloading (compile-time polymorphism / static polymorphism) allows multiple methods in the same class to share the same name but differ in the number, type, or order of parameters. **Q75.** In C++, what is the purpose of a destructor?

- B) To overload the constructor
- C) To release resources held by an object when it goes out of scope or is deleted

**Answer:** C
**Explanation:** A destructor is a special member function invoked automatically when an object's lifetime ends (goes out of scope, `delete` is called, or program exits). **Q76.** What is "data abstraction" in OOP?

- B) Grouping similar objects together
- C) Allowing a class to inherit from multiple classes
- D) Storing data in a database

**Answer:** A
**Explanation:** Abstraction is the OOP principle of hiding internal implementation details while exposing only the necessary interface to the user. It reduces complexity and increases efficiency. **Q77.** Which keyword prevents a class from being subclassed in Java/C#?

**Answer:** C
**Explanation:** In Java, the `final` keyword on a class prevents it from being subclassed. **Q78.** What is the output of the following C++ code?
```cpp
class Animal {
public:
    void speak() { cout << "Animal"; }
};
class Dog : public Animal {
public:
    void speak() { cout << "Dog"; }
};
int main() {
    Animal *a = new Dog();
    a->speak();
    return 0;
}
```
- A) Animal
- B) Dog
- C) Compilation error
- D) Runtime error

**Answer:** A
**Explanation:** Since `speak()` is not declared `virtual` in the base class, this is static binding (early binding). The pointer type is `Animal*`, so `Animal::speak()` is called regardless of the actual object being a `Dog`. If `virtual void speak()` were used, the output would be "Dog" (dynamic dispatch/late binding).

**Q79.** Which of the following is an example of runtime polymorphism in C++?
- A) Function overloading
- B) Operator overloading
- C) Template specialisation
- D) Virtual function overriding

**Answer:** D
**Explanation:** Runtime polymorphism (dynamic dispatch) occurs when the function to be called is determined at runtime based on the actual object type, not the pointer/reference type. This is achieved through virtual functions in C++. Function overloading, operator overloading, and templates are all resolved at compile time (static polymorphism).

**Explanation:** In C++, class members are `private` by default — they cannot be accessed from outside the class. This enforces encapsulation. In `struct`, the default access specifier is `public`. In Java, the default (no keyword) is package-private (accessible within the same package).

**Q81.** Which OOP pillar does the concept of "IS-A" relationship represent?

**Q82.** What is a "pure virtual function" in C++?
- A) A virtual function that has a body
- B) A function declared with `= 0` that must be overridden in derived classes, making the class abstract
- C) A static virtual function
- D) A function that cannot be called

**Answer:** B
**Explanation:** A pure virtual function is declared by assigning 0 to it: `virtual void draw() = 0;`. **Q83.** What is the difference between shallow copy and deep copy in OOP?
- A) Shallow copy copies only primitive types; deep copy copies objects
- B) Shallow copy copies references to objects (both point to the same memory); deep copy creates independent copies of all objects
- C) Deep copy is faster than shallow copy
- D) Both do exactly the same thing

**Answer:** B
**Explanation:** A shallow copy copies the object's field values — for reference types, it copies the reference (pointer), not the actual object. Deep copy is safer but slower and requires more memory.

### Part 9: Operating Systems (Q84–Q86)

**Q84.** What is the purpose of the "page table" in an operating system?
- A) To store a list of running processes
- B) To map virtual (logical) addresses to physical memory addresses
- C) To manage the file system
- D) To schedule CPU tasks

**Answer:** B
**Explanation:** A page table is a data structure maintained by the OS for each process that maps virtual page numbers to physical frame numbers. When a process accesses a virtual address, the Memory Management Unit (MMU) uses the page table to translate it to the actual physical address. This enables virtual memory and process isolation.

---

**Q85.** What is the difference between "preemptive" and "non-preemptive" scheduling?
- A) Preemptive scheduling does not allow CPU to be taken from a process; non-preemptive does
- B) Preemptive scheduling allows the OS to forcibly take the CPU from a running process; non-preemptive allows a process to run until it voluntarily releases the CPU
- C) Both are the same
- D) Non-preemptive scheduling is always more efficient

**Answer:** B
**Explanation:** In preemptive scheduling (e.g., Round Robin, SRTF), the OS can interrupt and suspend a running process to allocate the CPU to another process (e.g., higher priority). In non-preemptive scheduling (e.g., FCFS, SJF non-preemptive), once a process is given the CPU, it runs until it completes or voluntarily blocks for I/O. Preemptive is more responsive but complex.

**Q86.** What is "thrashing" in the context of virtual memory?
- A) A CPU becoming too hot
- B) A situation where the OS spends more time swapping pages in and out than executing processes
- C) A disk becoming full
- D) A network being overloaded

**Answer:** B
**Explanation:** Thrashing occurs when a process (or the system) does not have enough physical memory frames to hold its working set — causing frequent page faults. The OS spends most of its time swapping pages between RAM and disk, severely degrading performance. Solutions include increasing RAM, reducing multiprogramming degree, or using working set models.

### Part 10: Networking (Q87–Q91)

**Q87.** What is the difference between a "router" and a "switch"?
