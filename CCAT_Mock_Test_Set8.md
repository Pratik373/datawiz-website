**Q1.** The politician's speech was full of _____ promises that the public could see right through.
- A) sincere
- B) hollow
- C) genuine
- D) decisive

**Answer:** B
**Explanation:** "Hollow promises" is a well-established idiom meaning promises that lack substance or sincerity — exactly what the context suggests ("the public could see right through"). "Sincere" and "genuine" are positive and contradictory. "Decisive" means firm, not deceptive.

**Q2.** Her arguments, though _____, lacked the empirical data needed to convince the panel.
- A) feeble
- B) incoherent
- C) compelling
- D) irrelevant

**Answer:** C
**Explanation:** The sentence contrasts the quality of arguments ("though _____") with a lack of evidence. The correct word must be positive (conceding the arguments had merit) while still being unable to convince. "Compelling" fits — the arguments were persuasive in form but unsupported by evidence. "Feeble" and "irrelevant" would not need the contrast introduced by "though."

**Q3.** Spot the error: "No sooner did he arrive, when everyone started clapping."
- A) No sooner did he arrive
- B) when everyone
- C) started clapping
- D) No error

**Answer:** B
**Explanation:** "No sooner…than" is the correct correlative conjunction. "When" should be replaced with "than." Correct sentence: "No sooner did he arrive than everyone started clapping." Confusing "No sooner…than" with "Hardly/Scarcely…when" is a common error.

**Q4.** Fill in the blank: The sky was overcast, and it looked as if it _____ rain heavily.
- A) will
- B) would
- C) shall
- D) can

**Answer:** B
**Explanation:** "It looked as if" introduces a hypothetical or apparent situation in past context — requiring a past modal. "Would" is the correct choice here. "Will" is present tense and cannot follow "looked as if." "Shall" is used for first person in formal contexts. "Can" expresses ability, not likelihood.

**Q5.** Choose the grammatically correct sentence:
- A) The data was collected from various sources.
- B) The data were collected from various sources.
- C) The datas were collected from various sources.
- D) Both A and B are correct depending on usage

**Answer:** D
**Explanation:** "Data" is technically the plural of "datum" and traditionally takes a plural verb ("data were"). However, in modern everyday English, "data" is increasingly treated as a mass noun with a singular verb ("data was"). Both are now accepted in informal and many professional contexts. In formal/academic settings, "data were" is preferred.

**Q44.** What is the difference between RAM and ROM?
- A) RAM is permanent; ROM is temporary
- B) RAM is volatile and loses data when power is off; ROM is non-volatile and retains data
- C) ROM can be written to repeatedly; RAM cannot

### Part 7: Data Structures (Q64–Q71)

**Q64.** What is the time complexity of deleting an element from the middle of an array (not linked list)?
- A) O(1)
- B) O(log n)
- C) O(n)
- D) O(n²)

**Answer:** C
**Explanation:** Deleting from the middle of an array requires shifting all subsequent elements one position to the left to fill the gap. In the worst case (deleting from the beginning), all n−1 elements must be shifted — O(n). This is why linked lists are preferred when frequent insertions/deletions in the middle are needed.

**Q65.** Which sorting algorithm is NOT comparison-based?
- A) Quick Sort
- B) Merge Sort
- C) Counting Sort
- D) Heap Sort

**Answer:** C
**Explanation:** Counting Sort is a non-comparison-based (integer) sorting algorithm that works by counting occurrences of each element and using that information to place elements in sorted order. It can achieve O(n+k) time complexity. Quick Sort, Merge Sort, and Heap Sort all rely on comparisons between elements.

**Q66.** What is a "deque" (Double-Ended Queue)?
- A) A stack that can hold double the elements
- B) A queue where insertion and deletion can happen at both ends
- C) A priority queue with two priority levels
- D) A sorted linked list

**Answer:** B
**Explanation:** A deque (pronounced "deck") is a data structure where elements can be inserted and deleted from both the front and rear. It generalises both stacks (rear insert/delete) and queues (rear insert, front delete). It is implemented using doubly linked lists or circular arrays.

**Q67.** In a hash table with chaining for collision resolution, what is the average time complexity of search?
- A) O(1) always
- B) O(n) always
- C) O(1 + load_factor) on average
- D) O(log n)

**Answer:** C
**Explanation:** With chaining, the average search time is O(1 + α), where α = n/m is the load factor (n = elements, m = table size). If the load factor is kept low (< 1) and the hash function distributes keys uniformly, the average case approaches O(1). Worst case is O(n) if all keys hash to the same bucket.

**Q68.** Which traversal of a tree visits all nodes at the current depth before moving to the next depth?
- A) Inorder
- B) Preorder
- C) Postorder
- D) Level-order (BFS)

**Answer:** D
**Explanation:** Level-order traversal (Breadth-First Search) visits all nodes at depth 0 (root), then depth 1, then depth 2, etc. It uses a queue. Inorder, Preorder, and Postorder are depth-first traversals that use a stack (implicit or explicit).

**Q69.** What is the minimum number of nodes in a binary tree of height h?
- A) h
- B) h + 1
- C) 2h
- D) 2h + 1

**Answer:** B
**Explanation:** A binary tree of height h has at minimum h + 1 nodes — this is a skewed tree where each node has only one child (essentially a linked list). Maximum nodes = 2^(h+1) − 1 (a perfect/complete binary tree).

**Q70.** Which of the following is true about Heap data structure?
- A) A heap is always a balanced BST
- B) Searching an element in a heap takes O(1) time
- C) The heap property ensures the root is either the maximum or minimum element
- D) Heaps support efficient insertion at the end only

**Answer:** C
**Explanation:** In a Max-Heap, the root is always the maximum element; in a Min-Heap, it's the minimum. A heap is a complete binary tree satisfying the heap property — it is NOT a BST (no ordering between left and right siblings). Searching a heap is O(n). Insertion is O(log n).

**Q71.** What is "dynamic programming" as applied to algorithms?
- A) Programming done at runtime
- B) A technique that solves complex problems by breaking them into overlapping subproblems and storing results to avoid recomputation
- C) Programming that adjusts algorithm complexity dynamically
- D) A technique for sorting dynamic data

**Answer:** B
**Explanation:** Dynamic Programming (DP) solves optimisation problems by breaking them into overlapping subproblems, storing (memoising) solutions to avoid redundant recomputation. Key examples: Fibonacci, Knapsack Problem, Longest Common Subsequence. It is different from divide-and-conquer (which has non-overlapping subproblems).

**Q82.** What is "inter-process communication" (IPC)?

- C) Communication between the CPU and RAM

**Answer:** B

**Q83.** What is "segmentation" in memory management?

**Q84.** What is the purpose of the "PCB" (Process Control Block)?

**Q85.** What is the "working set model" in virtual memory management?

- B) The set of pages that a process has recently referenced and is likely to need soon

**Answer:** B
**Explanation:** The working set of a process is the set of pages it has referenced during a recent time window (the working set window). The OS tries to keep all pages in a process's working set in physical memory. If the combined working sets exceed physical memory, thrashing occurs.

**Q87.** What is the difference between TCP and UDP?

- B) UDP is connection-oriented; TCP is connectionless

**Answer:** C
**Explanation:** TCP (Transmission Control Protocol) establishes a connection (3-way handshake), ensures reliable delivery with acknowledgements, retransmission, and ordering.
