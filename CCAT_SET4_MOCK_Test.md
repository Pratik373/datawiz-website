**Q1.** The verdict was delivered _____ the absence of key evidence.

- A. despite
- B. inspite
- C. in spite of
- D. regardless

**Answer: C — in spite of**
**Explanation:** "In spite of" is correct because it is a prepositional phrase that must be followed by a noun/noun phrase ("the absence of key evidence"). "Despite" alone also works grammatically, but "inspite" (without space) is incorrect spelling and is not a valid option.

- D. in

**Explanation:** "Taken aback" is a fixed phrasal expression meaning to be surprised, shocked, or caught off-guard. None of the other prepositions form a standard idiom with "taken" in this context.

**Answer: D — 77**

for i in range(n):
    j = 1
    while j < n:
        j = j * 2
```
- A. O(n)
- B. O(n²)
- C. O(n log n)
- D. O(log n)

**Answer: C — O(n log n)**
**Explanation:** The outer `for` loop runs n times. For each iteration, the inner `while` loop starts j at 1 and doubles it each time (j = 1, 2, 4, 8, ...) until j ≥ n — this runs O(log n) times. Total = n × log n = O(n log n).

**Q65.** What is the output of the following code?

**Q67.** What is the output of the following code?

union Data { int i; float f; };

```
- A. 10
- B. 3

**Explanation:** In a union, all members share the same memory location. After assigning `d.f = 3.14`, the float value overwrites the shared memory. Reading `d.i` now interprets the bits of the float 3.14 as an integer, producing a meaningless (garbage) value.

**Q68.** What is the purpose of `realloc()` in C?

- A. Reallocates memory to a new address always
- B. Resizes a previously allocated memory block
- C. Frees and re-allocates fresh zeroed memory
- D. Copies memory from one pointer to another

**Answer: B — Resizes a previously allocated memory block**
**Explanation:** `realloc(ptr, new_size)` resizes the memory block pointed to by `ptr` to `new_size` bytes, preserving the existing data up to the smaller of old and new sizes. It may return the same or a new address. Unlike `calloc`, it does not zero-initialize the new memory.

### PART 8: Object Oriented Programming — OOPs (Q74–Q83)

**Q74.** Which of the following best describes Object-Oriented Programming (OOP)?

- A. Programming using only functions
- B. Programming based on objects and classes
- C. Programming without data
- D. Programming using only loops

**Answer: B — Programming based on objects and classes**
**Explanation:** OOP is a programming paradigm that organizes software design around objects (instances of classes) rather than functions or logic alone. The four core pillars of OOP are: Encapsulation, Inheritance, Polymorphism, and Abstraction.

---

**Q75.** What is the main purpose of encapsulation in OOP?

- A. To increase memory usage
- B. To hide data and protect it from unauthorized access
- C. To create multiple objects
- D. To reduce the number of classes

**Answer: B**
**Explanation:** Encapsulation bundles data (attributes) and methods (functions) into a single unit (class) and restricts direct access to internal data using access specifiers (private/protected). This protects the object's state from unintended external interference, promoting data integrity.

---

**Q76.** Which access specifier allows class members to be accessed only within the same class?

- A. public
- B. protected
- C. private
- D. global

**Answer: C — private**
**Explanation:** `private` members are accessible only within the class they are defined in — not by derived classes or outside code. `protected` allows derived class access. `public` allows unrestricted access. `global` is not an access specifier in C++.

---

**Q77.** Which concept allows a derived class to acquire properties of a base class?

- A. Polymorphism
- B. Inheritance
- C. Abstraction
- D. Encapsulation

**Answer: B — Inheritance**
**Explanation:** Inheritance is the OOP mechanism where a derived (child) class acquires the properties and behaviors (attributes and methods) of a base (parent) class. It promotes code reuse and establishes an "is-a" relationship between classes.

---

**Q78.** Which of the following is an example of compile-time polymorphism?

- A. Virtual Function
- B. Function Overriding
- C. Function Overloading
- D. Dynamic Binding

**Answer: C — Function Overloading**
**Explanation:** Compile-time (static) polymorphism is resolved at compile time. Function Overloading (same name, different parameters in the same class) is the classic example. Virtual functions, function overriding, and dynamic binding are all examples of runtime (dynamic) polymorphism.

---

**Q79.** What is the purpose of a constructor in C++?

- A. To destroy objects
- B. To initialize objects
- C. To overload operators
- D. To inherit classes

**Answer: B — To initialize objects**
**Explanation:** A constructor is a special member function automatically called when an object is created. Its primary purpose is to initialize the object's data members to valid initial values. It has the same name as the class and no return type.

**Q80.** What will be the output of the following C++ code?
```cpp
class Test {

    Test() { cout << "Constructor Called"; }

int main() {
    Test t;
    return 0;
}
```
- A. Error
- B. Constructor Called
- C. Destructor Called
- D. No Output

**Answer: B — Constructor Called**
**Explanation:** When the object `t` of class `Test` is created, the constructor `Test()` is automatically invoked, printing "Constructor Called". Since no destructor output is defined, only the constructor message appears.

**Q81.** Which keyword is used to refer to the current object inside a class in C++?

- A. current
- B. object
- C. self
- D. this

**Answer: D — this**
**Explanation:** `this` is a pointer available inside every non-static member function that points to the current object (the object that invoked the function). It is used to resolve naming conflicts between data members and parameters, and to return the current object from a function.

**Q82.** What will be the output of the following C++ program?
```cpp
class Base {
public:
    virtual void show() { cout << "Base"; }
};
class Derived : public Base {
public:
    void show() { cout << "Derived"; }
};
int main() {
    Base *b;
    Derived d;
    b = &d;
    b->show();
    return 0;
}
```
- A. Base
- B. Derived
- C. BaseDerived
- D. Error

**Answer: B — Derived**
**Explanation:** Because `show()` is declared `virtual` in the base class, the call `b->show()` is resolved at runtime based on the actual type of the object (`Derived`), not the type of the pointer (`Base*`). This is runtime polymorphism / dynamic dispatch → prints "Derived".

**Q83.** Which of the following statements about abstract classes is correct?

- A. An abstract class can be instantiated directly
- B. An abstract class cannot contain functions
- C. An abstract class contains at least one pure virtual function
- D. Abstract classes are not supported in C++

**Answer: C**
**Explanation:** An abstract class in C++ is one that contains at least one pure virtual function (declared with `= 0`). Abstract classes cannot be instantiated directly — they serve as base classes for derived classes that must implement all pure virtual functions. They can contain both pure virtual and regular functions.

**Q84.** Which operating system is commonly used in servers?

- A. Linux
- B. MS Paint
- C. Android Studio
- D. Photoshop

**Answer: A — Linux**
**Explanation:** Linux is the dominant OS for servers due to its stability, security, open-source nature, and low resource overhead. MS Paint is a graphics application, Android Studio is an IDE, and Photoshop is image editing software — none are operating systems.

**Q85.** Which of the following supports multitasking?

- A. Modern Operating Systems
- B. BIOS
- C. Compilers
- D. Assemblers

**Answer: A — Modern Operating Systems**
**Explanation:** Multitasking — running multiple processes concurrently — is managed by the OS through CPU scheduling. BIOS only initializes hardware at startup; Compilers translate source code to machine code; Assemblers convert assembly language to machine code. None of these support multitasking.

---

**Q86.** Which scheduling algorithm may lead to starvation?

- A. FCFS
- B. Round Robin
- C. Priority Scheduling
- D. FIFO

**Answer: C — Priority Scheduling**
**Explanation:** In Priority Scheduling, a low-priority process may never get CPU time if higher-priority processes keep arriving — this is called starvation. Round Robin avoids starvation by giving each process a fixed time quantum. FCFS and FIFO don't cause starvation as they serve processes in arrival order.

---

**Q87.** Which memory management scheme suffers from external fragmentation?

- A. Paging
- B. Segmentation
- C. Virtual Memory
- D. Cache Memory

**Answer: B — Segmentation**
**Explanation:** Segmentation divides memory into variable-sized segments, which over time creates gaps of unusable free memory between allocated segments — this is external fragmentation. Paging uses fixed-size pages/frames, so it eliminates external fragmentation (but suffers from internal fragmentation).

---

**Q88.** Semaphores are mainly used for:

- A. File Compression
- B. Synchronization
- C. Memory Allocation
- D. Routing

**Answer: B — Synchronization**
**Explanation:** Semaphores are integer-based synchronization primitives (proposed by Dijkstra) used to control access to shared resources in concurrent/parallel programming. They solve the critical section problem and prevent race conditions. They are not used for file compression, memory allocation, or network routing.

### PART 10: Networking (Q89–Q93)

**Q89.** Which layer of the OSI model is responsible for end-to-end communication?

- A. Transport Layer
- B. Network Layer
- C. Data Link Layer
- D. Physical Layer

**Answer: A — Transport Layer**
**Explanation:** The Transport Layer (Layer 4) of the OSI model is responsible for end-to-end communication between source and destination hosts. It provides reliable data transfer (TCP) or fast connectionless transfer (UDP), error detection, flow control, and segmentation/reassembly.

---

**Q90.** Which protocol is used to automatically assign IP addresses?

- A. DNS
- B. DHCP
- C. FTP
- D. ARP

**Answer: B — DHCP**
**Explanation:** DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses and other network configuration parameters (subnet mask, gateway, DNS server) to devices on a network. DNS translates domain names to IPs; FTP transfers files; ARP resolves IP addresses to MAC addresses.

---

**Q91.** Which topology provides high reliability due to multiple paths?

- A. Bus
- B. Star
- C. Ring
- D. Mesh

**Answer: D — Mesh**
**Explanation:** In a Mesh topology, every device is connected to every other device via dedicated links, providing multiple paths for data. If one link fails, data can be routed through alternate paths — making it highly reliable and fault-tolerant. It is used in WANs and critical network infrastructure.

---

**Q92.** Which port is generally used by HTTP?

- A. 21
- B. 25
- C. 80
- D. 443

**Answer: C — 80**
**Explanation:** HTTP (HyperText Transfer Protocol) uses port 80 by default for web traffic. HTTPS (secure HTTP) uses port 443. Port 21 is used by FTP (File Transfer Protocol). Port 25 is used by SMTP (Simple Mail Transfer Protocol).

---

**Q93.** Which layer converts bits into signals?

- A. Physical Layer
- B. Network Layer
- C. Session Layer
- D. Application Layer

**Answer: A — Physical Layer**
**Explanation:** The Physical Layer (Layer 1) of the OSI model is responsible for the transmission of raw bits over a physical medium (copper wire, fiber optic, radio waves). It handles conversion of digital bits into electrical, optical, or radio signals and defines hardware standards like voltage levels and cable types.

**Q96.** Which type of data includes images, videos, and social media posts?

- A. Structured Data
- B. Numerical Data
- C. Relational Data
- D. Unstructured Data

**Answer: D — Unstructured Data**
**Explanation:** Unstructured data has no predefined format or schema. Images, videos, audio files, emails, and social media posts are classic examples. Structured data fits neatly into rows and columns (e.g., SQL databases). About 80–90% of all data generated today is unstructured.

HDFS handles storage; YARN manages resources; Hive provides SQL-like querying.
