**Q1.** The administration's response to the disaster was widely criticised for being _____.

**Q14.** The difference between compound interest and simple interest on ₹10,000 at 10% per annum for 2 years is:
- A) ₹100
- B) ₹150
- C) ₹200
- D) ₹250

**Answer:** A
**Explanation:** SI = P×R×T/100 = 10000×10×2/100 = ₹2,000. CI = P[(1+r)ⁿ−1] = 10000[(1.1)²−1] = 10000×0.21 = ₹2,100. Difference = 2100 − 2000 = ₹100.

**Q49.** What is the time complexity of the following algorithm?
```
for i = 1 to n:
    for j = 1 to n:
        print(i * j)
```
- A) O(n)
- B) O(n log n)
- C) O(n²)
- D) O(2n)

**Answer:** C
**Explanation:** The outer loop runs n times and the inner loop also runs n times for each iteration of the outer loop, resulting in n × n = n² total operations. Time complexity = O(n²).

### Part 9: Operating Systems (Q82–Q86)

**Q82.** What is the difference between a process and a thread?
- A) A process is lighter than a thread
- B) A thread is an independent program; a process is a sub-unit of a thread
- C) A process is an independent program execution unit; a thread is the smallest execution unit within a process sharing its resources

**Answer:** C
**Explanation:** A process is an independent program in execution with its own memory space. **Q83.** What is "starvation" in operating systems?

- B) When a low-priority process is indefinitely denied the resources it needs
- C) When the OS runs out of memory
- D) When all processes are waiting for each other

**Answer:** B
**Explanation:** Starvation occurs when a process is repeatedly denied access to resources because higher-priority processes continuously preempt it. It is a problem in priority scheduling algorithms. "Aging" (gradually increasing a process's priority the longer it waits) is a common solution.

**Q84.** Which page replacement algorithm always replaces the page that will not be used for the longest time in the future?
- A) FIFO
- B) LRU
- C) Optimal (OPT)
- D) MRU

**Answer:** C
**Explanation:** The Optimal (OPT or Bélády's) algorithm replaces the page that will not be referenced for the longest time in the future. **Q85.** What is the purpose of a "mutex" in concurrent programming?

- B) To ensure that only one thread at a time accesses a critical section
- C) To create multiple threads simultaneously

**Answer:** B
**Explanation:** A mutex (Mutual Exclusion lock) is a synchronisation primitive that ensures only one thread can access a critical section (shared resource) at a time. A thread "locks" the mutex before entering the critical section and "unlocks" it after exiting, preventing race conditions.

- B) To prevent packets from circulating forever by limiting the number of hops

**Answer:** B
**Explanation:** The TTL field in an IP header specifies the maximum number of router hops a packet can travel before being discarded.
