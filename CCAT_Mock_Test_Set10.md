**Q3.** Spot the error: "The data pertaining to the survey have been collected from three hundred respondents and was analysed using SPSS software."
- A) The data pertaining to the survey
- B) have been collected from three hundred respondents
- C) and was analysed using SPSS software
- D) No error

**Answer:** C
**Explanation:** "The data … have been collected … and was analysed" is inconsistent — if "have been" (plural/formal) is used for "collected," the same subject-verb agreement must apply to "analysed." **Q46.** What is the worst-case time complexity of Quick Sort?
- A) O(n log n)
- B) O(n²)
- C) O(n)
- D) O(log n)

**Answer:** B
**Explanation:** Quick Sort's worst case is O(n²), occurring when the pivot is always the smallest or largest element (e.g., already sorted array with the first element as pivot). **Q47.** What is "tail recursion"?

- B) OLTP (Online Transaction Processing) handles high-volume transactional queries; OLAP (Online Analytical Processing) handles complex analytical queries on large historical datasets
- C) OLAP is for real-time systems; OLTP for historical analysis
- D) OLTP uses NoSQL; OLAP uses SQL

**Answer:** B
**Explanation:** OLTP systems support day-to-day transactions — high volume, short queries, CRUD operations, low latency (e.g., banking, e-commerce). OLAP systems support business intelligence and analytics — complex queries on large historical datasets, aggregations, read-heavy (e.g., data warehouses, Redshift, BigQuery). OLTP is normalised (reduces redundancy); OLAP is denormalised (star/snowflake schemas) for query performance.

**Q93.** What is "regularisation" in machine learning and what does it prevent?
- A) Making a model more complex to fit training data better
- B) A technique that adds a penalty term to the loss function to discourage overly complex models, preventing overfitting
- C) Standardising the input data
- D) Regularising the learning rate

**Answer:** B
**Explanation:** Regularisation adds a penalty to the model's loss function for complexity (large weights). L1 regularisation (Lasso) adds the sum of absolute weights — can force some weights to exactly zero (feature selection). L2 regularisation (Ridge) adds the sum of squared weights — reduces all weights towards zero but rarely to exactly zero. Both prevent overfitting by constraining model complexity.

**Q94.** What is the "vanishing gradient" problem in deep neural networks?
- A) When the network loses track of the training data
- B) When gradients become exponentially small as they propagate back through many layers, making earlier layers learn very slowly
- C) When the learning rate drops to zero
- D) When the network has too few parameters

**Answer:** B
**Explanation:** In deep networks, backpropagation multiplies many gradient values together through the chain rule. With activation functions like sigmoid or tanh (outputs bounded 0–1 or −1 to 1), repeated multiplication causes gradients to shrink exponentially — becoming near-zero in early layers. Solutions: ReLU activation (avoids squashing positives), batch normalisation, residual connections (ResNets), and careful weight initialisation.

**Q95.** What is the difference between "bagging" and "boosting" in ensemble learning?
- A) Bagging and boosting are the same technique
- B) Bagging trains multiple models in parallel on different data subsets (reducing variance); boosting trains models sequentially where each corrects the previous model's errors (reducing bias)
- C) Bagging uses neural networks; boosting uses decision trees only
- D) Bagging is for classification only; boosting for regression only

**Answer:** B
**Explanation:** Bagging (Bootstrap Aggregating) creates multiple models trained on random subsets (with replacement) in parallel and combines predictions by averaging/voting — reduces variance, prevents overfitting (e.g., Random Forest). Boosting trains models sequentially — each new model focuses on misclassified examples — reduces bias (e.g., AdaBoost, Gradient Boosting, XGBoost). Boosting often achieves higher accuracy but is more prone to overfitting.

**Q96.** What is a "data lake" and how does it differ from a "data warehouse"?
- A) They are synonymous terms
- B) A data lake stores raw, unprocessed data in any format (structured/unstructured) at low cost; a data warehouse stores processed, structured data optimised for analytics
- C) A data warehouse is for real-time data; a data lake for historical data
- D) Data lakes only store images and videos

**Answer:** B
**Explanation:** Data Lake: stores massive amounts of raw data in native format (structured, semi-structured, unstructured — JSON, CSV, images, logs) at low cost (e.g., S3, HDFS) — schema-on-read. Data Warehouse: stores cleaned, processed, structured data in a predefined schema — schema-on-write — optimised for SQL analytics (e.g., Redshift, BigQuery, Snowflake). Modern architectures use data lakehouses (e.g., Delta Lake) combining both.

**Q97.** What is "attention mechanism" in deep learning, particularly in transformers?
- A) A technique to keep the model focused on training
- B) A mechanism that allows the model to weigh the importance of different parts of the input sequence when producing each output, enabling it to focus on relevant context
- C) A regularisation technique
- D) A method to speed up GPU computations

**Answer:** B
**Explanation:** The attention mechanism computes a weighted sum of input representations where the weights (attention scores) indicate how relevant each input position is for the current output. Self-attention in Transformers allows each token to attend to all other tokens in the sequence. This enables capturing long-range dependencies without recurrence (unlike RNNs/LSTMs) and is the foundation of models like BERT, GPT, and the entire large language model (LLM) paradigm.

**Q98.** What is "MapReduce" in the context of big data processing?
- A) A database query language
- B) A programming model for processing large datasets in parallel across a distributed cluster — Map phase applies a function to each element; Reduce phase aggregates results
- C) A real-time stream processing framework
- D) A machine learning algorithm

**Answer:** B
**Explanation:** MapReduce (popularised by Google, implemented in Hadoop) has two phases: Map — processes input key-value pairs and emits intermediate key-value pairs (e.g., for each word, emit (word, 1)); Reduce — aggregates all values for the same key (e.g., sum all counts for each word → word frequency). The framework handles parallel distribution, fault tolerance, and data shuffling across the cluster.

**Q99.** What is "model interpretability" and why is it important?
- A) The speed at which a model makes predictions
- B) The degree to which humans can understand and explain how a model makes its predictions — important for trust, debugging, compliance, and identifying bias
- C) The ability of a model to handle multiple languages
- D) The model's performance on test data

**Answer:** B
**Explanation:** Model interpretability (explainability) refers to how well a human can understand a model's decision-making process. It is critical in: healthcare (why was this diagnosis made?), finance (why was a loan rejected?), legal compliance (GDPR's right to explanation), and bias detection. Techniques include LIME (Local Interpretable Model-Agnostic Explanations), SHAP (SHapley Additive exPlanations), and inherently interpretable models (decision trees, linear regression).

**Q100.** What is the role of "Zookeeper" in distributed big data systems like Hadoop/Kafka?
- A) It monitors wild animals in a zoo network
- B) A centralised coordination service that manages distributed configuration, naming, synchronisation, and group services for distributed applications
- C) A data storage layer in HDFS
- D) A machine learning framework for distributed training

**Answer:** B
**Explanation:** Apache ZooKeeper is a distributed coordination service providing: distributed configuration management, naming (registry), distributed synchronisation (locks, barriers, elections), and group membership services. It enables distributed systems like HBase, Kafka (broker leader election), and Hadoop (NameNode HA) to coordinate without each implementing complex distributed protocols themselves. ZooKeeper uses a tree-like data model (znodes) similar to a filesystem.
